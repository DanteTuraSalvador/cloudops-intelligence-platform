## Monitoring Your Deployment

### Check Lambda Functions:
```cmd
aws lambda list-functions --region ap-southeast-2 --query "Functions[?contains(FunctionName, 'cloudops')].FunctionName"
```

### Check API Gateway:
```cmd
aws apigatewayv2 get-apis --region ap-southeast-2 --query "Items[?Name=='cloudops-api'].{Name:Name,ApiId:ApiId,Endpoint:ApiEndpoint}"
```

### Check EventBridge Rules:
```cmd
aws events list-rules --region ap-southeast-2 --query "Rules[?contains(Name, 'cloudops')].Name"
```

### Check Data Collection:
Wait 5 minutes, then:
```cmd
aws dynamodb scan --table-name cloudops-metrics --limit 5 --region ap-southeast-2
```

### View CloudWatch Logs:
```cmd
aws logs tail /aws/lambda/cloudops-api-gateway --region ap-southeast-2 --follow
```

## Troubleshooting

### If deployment fails:

1. **Check AWS credentials**:
   ```cmd
   aws sts get-caller-identity
   ```

2. **Verify Free Tier eligibility**:
   - Go to AWS Console ‚Üí Billing ‚Üí Free Tier
   - Ensure you're within 12 months of account creation

3. **Check IAM permissions**:
   - Your AWS user needs permissions to create IAM roles, Lambda, API Gateway, EventBridge

4. **Manual deployment**:
   - Follow steps in `deployment/DEPLOYMENT_GUIDE.txt`
